# Philosophy

The goal is not to collect plugins. The goal is FLOW.

Good tooling should answer three questions quickly:

1. Where am I?
2. How do I open the thing I need?
3. How do I move between shell, editor, Git, and AI without friction?

This setup starts small on purpose. First we build muscle memory: terminal panes, project selection, Neovim navigation, file search, text search, and OpenCode in the right folder.

After that foundation feels natural, then we can add heavier IDE features like LSP, formatters, test runners, and deeper AI integration. Not before. More tools without understanding is just more noise.
